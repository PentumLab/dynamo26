DYNAMO-26 PIPELINE SPEC v1.0
Deterministischer Executor-Entwurf
Sprache: Deutsch
Ziel: Ein vollständig prüfbarer, quellengetriebener Tageslauf zur Bewertung von WM-2026-Titelstabilität.

============================================================
0. GRUNDPRINZIP
============================================================

Diese Spezifikation beschreibt keinen Fließtext-Prompt und keine lose Checkliste.
Sie beschreibt einen deterministischen Ablauf.

Jeder Schritt hat:
- Eingabe
- Pflichtaktion
- Rückgabeobjekt
- Gate
- Audit

Ein Schritt darf erst verlassen werden, wenn sein Rückgabeobjekt vollständig ist.
Ein späterer Schritt darf keine Daten erfinden, die ein früherer Schritt nicht geliefert hat.

============================================================
1. FESTE REGELN
============================================================

RULE_01:
Ohne aktuellen Quellenlauf gibt es keine aktuelle Prognose.

RULE_02:
Ein gespeicherter Vorzustand ist nur ein Vorzustand.
Er ist keine aktuelle Quelle.

RULE_03:
Wahrscheinlichkeiten dürfen nur ausgegeben werden, wenn:
- aktuelle Quellen geprüft wurden,
- Quellenqualität ausreichend ist,
- keine blockierenden Konflikte offen sind,
- Teamzustände berechnet wurden,
- Pfadsimulation abgeschlossen wurde,
- Wahrscheinlichkeiten auf 100 Prozent normalisiert wurden,
- Audit bestanden wurde.

RULE_04:
T7 bleibt als Modellinput vollständig verboten.
Externe Prognosen, Projektionen, Buchmacherquoten, Elo-/Ranking-Prognosen, FIFA-Ranking als Prognoseersatz oder Medien-Wahrscheinlichkeiten dürfen nicht gezielt gesucht, zitiert, gespeichert, zur Kalibrierung verwendet oder anderweitig als Modellinformation verwertet werden.
Wenn eine Websuche solche Treffer liefert, werden sie ignoriert und nicht gezielt geöffnet.
Wenn eine ansonsten zulässige T0-T6-Quelle zufällig einen T7-Abschnitt enthält, wird ausschließlich dieser T7-Teil als ignored_t7_segment dokumentiert und ignoriert. Die unabhängig belegten Nicht-T7-Fakten derselben Quelle dürfen weiter verwendet werden, sofern sie durch zulässige Quellenqualität und Audit gedeckt sind.
Ein ignored_t7_segment darf keinen NOT_CURRENT-Status auslösen und keine Kausalkette, Indexwirkung, Wahrscheinlichkeit oder Output-Formulierung beeinflussen.
Legacy-SRC_T7-Einträge aus älteren Prior-States dürfen nicht als aktive Quellen übernommen, aktualisiert oder ausgewertet werden.
Falls sie zur Nachvollziehbarkeit erwähnt werden müssen, dann nur als ignorierte Legacy-Information ohne Modellwirkung.

RULE_05:
Social-Media-Signale dürfen ohne Bestätigung keinen Fakt setzen.

RULE_06:
Fehlende geschlossene Premiumdaten blockieren den Lauf nicht automatisch.
Sie erzeugen Unsicherheitsflags und können die Konfidenz reduzieren.

RULE_07:
Jede finale OUTPUT-Aussage braucht eine Kausalkette:
Ursache -> Mechanismus -> Effekt -> Bedingung

RULE_08:
Am Ende müssen vier Hauptlauf-Artefakte erzeugt werden:
- deutsche HTML-Ausgabe
- englische HTML-Ausgabe
- Prior-State-TXT für den nächsten Lauf
- kurze Änderungs-TXT

RULE_09:
Wenn Pflichtartefakte fehlen, darf keine finale Antwort ausgeliefert werden.

============================================================
3. INDEXFORMELN
============================================================

Alle Indexwerte liegen zwischen 0 und 100.

SCI = Structural Capacity Index
TBI = Tournament Body Index
CSI = Contextual Strain Index
AMCI = Adaptive Match Control Index
SAI = Shock Absorption Index
FQ = Form Quality Index

CSI ist negativ gerichtet.
Höherer CSI bedeutet höhere Belastung.

DYNAMO_STRENGTH:
D = 0.22*SCI + 0.20*TBI + 0.18*AMCI + 0.16*SAI + 0.14*FQ - 0.10*CSI

CRASH_RISK:
CrashRisk =
0.18*low_chance_creation
+ 0.17*high_transition_exposure
+ 0.15*thin_squad_risk
+ 0.14*set_piece_weakness
+ 0.13*contextual_strain_pressure
+ 0.12*psychological_fragility
+ 0.11*path_difficulty

INDEX_UPDATE:
index_new = (1 - lambda) * index_old + lambda * observed_signal

LAMBDA_BOUNDS:
min = 0.02
max = 0.45

function RUN_PIPELINE(input):

  context = INIT_RUN(input)

  model = LOAD_MODEL(context)

  prior_state = LOAD_PRIOR_STATE(input.prior_state_file)

  source_plan = BUILD_SOURCE_PLAN(model, prior_state)

  source_run = FETCH_CURRENT_SOURCES(source_plan)

  if source_run.completed == false:
      context.output_status = "NOT_CURRENT"
      output = RENDER_NOT_CURRENT(context, source_run)
      artifacts = EXPORT_ARTIFACTS(context, output, prior_state, empty_changes)
      artifact_audit = AUDIT_ARTIFACTS(artifacts)
      finalization = FINALIZE_RUN(context, artifacts, artifact_audit)
      return DELIVER(context, output, artifacts, artifact_audit, finalization)

  classified_sources = CLASSIFY_SOURCES(source_run)

  source_quality = AUDIT_SOURCE_QUALITY(classified_sources)

  bias_result = AUDIT_BIAS(classified_sources, source_quality)

  status_gate = DETERMINE_OUTPUT_STATUS(source_run, source_quality)

  if status_gate.output_status == "NOT_CURRENT":
      context.output_status = "NOT_CURRENT"
      output = RENDER_NOT_CURRENT(context, status_gate)
      artifacts = EXPORT_ARTIFACTS(context, output, prior_state, empty_changes)
      artifact_audit = AUDIT_ARTIFACTS(artifacts)
      finalization = FINALIZE_RUN(context, artifacts, artifact_audit)
      return DELIVER(context, output, artifacts, artifact_audit, finalization)

  entities = RESOLVE_ENTITIES(classified_sources, prior_state)

  signals = EXTRACT_EVENT_SIGNALS(classified_sources, entities, prior_state)

  group_stage_gate = CHECK_GROUP_STAGE_FINAL_QUALIFICATION(source_run, classified_sources, prior_state)

  if group_stage_gate.required == true and group_stage_gate.passed == false:
      context.output_status = "NOT_CURRENT"
      output = RENDER_NOT_CURRENT(context, group_stage_gate)
      artifacts = EXPORT_ARTIFACTS(context, output, prior_state, empty_changes)
      artifact_audit = AUDIT_ARTIFACTS(artifacts)
      finalization = FINALIZE_RUN(context, artifacts, artifact_audit)
      return DELIVER(context, output, artifacts, artifact_audit, finalization)

  bracket_state = UPDATE_KNOCKOUT_BRACKET_STATE(source_run, classified_sources, prior_state, group_stage_gate)

  upcoming_matches = BUILD_UPCOMING_MATCH_PLAN(bracket_state, source_run)

  pre_match_source_plan = BUILD_PRE_MATCH_SOURCE_PLAN(upcoming_matches, prior_state)

  pre_match_source_run = FETCH_PRE_MATCH_SOURCES(pre_match_source_plan)

  pre_match_classified_sources = CLASSIFY_SOURCES(pre_match_source_run)

  pre_match_source_quality = AUDIT_SOURCE_QUALITY(pre_match_classified_sources)

  pre_match_dossiers = BUILD_PRE_MATCH_DOSSIERS(
      upcoming_matches,
      pre_match_classified_sources,
      prior_state
  )

  pre_match_audit = AUDIT_PRE_MATCH_DOSSIERS(
      upcoming_matches,
      pre_match_source_run,
      pre_match_source_quality,
      pre_match_dossiers
  )

  completeness = CHECK_INPUT_COMPLETENESS(
      source_run,
      source_quality,
      signals,
      pre_match_audit
  )

  context.output_status = DETERMINE_OUTPUT_STATUS(source_run, source_quality, completeness)

  if context.output_status == "NOT_CURRENT":
      output = RENDER_NOT_CURRENT(context, completeness)
      artifacts = EXPORT_ARTIFACTS(context, output, prior_state, empty_changes)
      artifact_audit = AUDIT_ARTIFACTS(artifacts)
      finalization = FINALIZE_RUN(context, artifacts, artifact_audit)
      return DELIVER(context, output, artifacts, artifact_audit, finalization)

  normalized_signals = NORMALIZE_SIGNALS(signals)

  team_states = BUILD_TEAM_STATES(prior_state, entities)

  event_update_result = APPLY_EVENT_UPDATES(team_states, normalized_signals)

  team_states = RECALCULATE_INDICES(team_states, event_update_result)

  team_states = CALCULATE_DYNAMIC_STRENGTH(team_states)

  matchup_matrix = BUILD_MATCHUP_MATRIX(team_states, pre_match_dossiers)

  path_result = SIMULATE_PATHS(matchup_matrix, team_states, bracket_state)

  probability_result = NORMALIZE_PROBABILITIES(path_result, bracket_state)

  team_states = ATTACH_PROBABILITIES(team_states, probability_result, bracket_state)

  team_states = CALCULATE_CRASH_RISK(team_states, path_result)

  causal_chains = GENERATE_CAUSAL_CHAINS(team_states, signals, probability_result)

  audit = AUDIT_MODEL(
      context,
      source_run,
      source_quality,
      bias_result,
      signals,
      team_states,
      probability_result,
      bracket_state,
      causal_chains,
      pre_match_dossiers,
      pre_match_audit
  )

  correction_count = 0

  while audit.passed == false and correction_count < 3:
      correction = SELF_CORRECT(audit)
      APPLY_CORRECTION(correction)
      audit = AUDIT_MODEL(...)
      correction_count = correction_count + 1

  if audit.passed == false:
      context.output_status = DOWNGRADE_STATUS(audit)

  output = RENDER_OUTPUT(
      context,
      source_run,
      source_quality,
      bias_result,
      team_states,
      probability_result,
      causal_chains,
      audit,
      pre_match_dossiers,
      pre_match_audit
  )

  changes = BUILD_CHANGE_SUMMARY(prior_state, team_states, signals)

  artifacts = EXPORT_ARTIFACTS(context, output, team_states, probability_result, causal_chains, pre_match_dossiers, changes)

  artifact_audit = AUDIT_ARTIFACTS(artifacts)

  if artifact_audit.passed == false:
      artifacts = EXPORT_ARTIFACTS(context, output, team_states, probability_result, causal_chains, pre_match_dossiers, changes)
      artifact_audit = AUDIT_ARTIFACTS(artifacts)

  finalization = FINALIZE_RUN(context, artifacts, artifact_audit, audit)

  return DELIVER(context, output, artifacts, artifact_audit, finalization)

function INIT_RUN(input):
  required actions:
    - run_date setzen
    - run_mode setzen
    - output_status = UNKNOWN setzen
    - audit_passed = false setzen
    - artifact_audit_passed = false setzen
  returns:
    RunContext

function LOAD_MODEL(context):
  required actions:
    - Indizes laden
    - Formeln laden
    - Quellenregeln laden
    - Biasregeln laden
    - Update-Regeln laden
    - Artefaktregeln laden
  returns:
    ModelConfig

function LOAD_PRIOR_STATE(prior_state_file):
  required actions:
    - Datei prüfen
    - created_at_run_date extrahieren
    - Wahrscheinlichkeiten extrahieren
    - Indexwerte extrahieren
    - Crash-Risiken extrahieren
    - Unsicherheitsflags extrahieren
  if no prior_state_file:
    - exists = false
  returns:
    PriorState

function BUILD_SOURCE_PLAN(model, prior_state):
  required source classes:
    - T0 official tournament core
    - T1 official team/federation information
    - T3 public match and lineup data
    - T4 venue, weather, travel, altitude, rest context
    - T5 reputable news
    - T6 social warning only
  forbidden source classes:
    - T7 external forecasts, projections, bookmaker odds, Elo-/ranking-based title predictions, media probabilities
  returns:
    SourcePlan

function FETCH_CURRENT_SOURCES(source_plan):
  required actions:
    - T0 core abrufen
    - T1 team sources abrufen
    - T3 public data abrufen
    - T4 context data abrufen
    - T5 reputable news abrufen
    - T6 nur als Warnsignal prüfen
    - T7 nicht gezielt suchen, zitieren, speichern, kalibrieren oder verwenden
    - zufällige T7-Abschnitte in ansonsten zulässigen Quellen als ignored_t7_segment dokumentieren und aus allen Fakten-/Signal-/Modellfeldern ausschließen
  returns:
    SourceRun

function CLASSIFY_SOURCES(source_run):
  required actions:
    - jede Quelle einem Tier zuordnen
    - jede Aussage als DATA markieren
    - Fakten von Interpretation trennen
    - Social Media ohne Bestätigung als warning_only markieren
  returns:
    list of SourceRecord

function AUDIT_SOURCE_QUALITY(classified_sources):
  required actions:
    - credibility_score berechnen
    - freshness_score berechnen
    - independence_score berechnen
    - specificity_score berechnen
    - actionability_score berechnen
    - total_score berechnen
    - Konflikte trennen in blocking und non_blocking
    - fehlende Daten trennen in critical und non_blocking
  returns:
    SourceQualityResult

function AUDIT_BIAS(classified_sources, source_quality):
  required checks:
    - recency bias
    - star bias
    - market bias
    - social hype bias
    - injury rumor bias
    - english language bias
    - regional undercoverage
    - closed data bias
  returns:
    BiasResult

function DETERMINE_OUTPUT_STATUS(source_run, source_quality, completeness optional):
  if source_run.completed == false:
      return NOT_CURRENT
  if source_quality.total_score < 60:
      return NOT_CURRENT
  if source_quality.unresolved_blocking_conflicts not empty:
      return NOT_CURRENT
  if source_quality.critical_missing_data not empty:
      return NOT_CURRENT
  if source_quality.total_score >= 75:
      return CURRENT
  if source_quality.total_score >= 60:
      return DEGRADED

function RESOLVE_ENTITIES(classified_sources, prior_state):
  required actions:
    - Teamnamen normalisieren
    - Spielernamen normalisieren
    - Spielorte normalisieren
    - Teamstatus prüfen
    - nicht qualifizierte Teams aus aktivem Modelluniversum entfernen
    - Wenn die Gruppenphase abgeschlossen ist, darf ein Team nur dann im aktiven K.o.-Modelluniversum bleiben, wenn CHECK_GROUP_STAGE_FINAL_QUALIFICATION es als qualifiziert bestätigt hat.
    - Wenn die Gruppenphase noch nicht abgeschlossen ist, dürfen Drittplatzierte nur als provisional/contender markiert werden; sie dürfen nicht endgültig als qualifiziert oder ausgeschieden behandelt werden.
  returns:
    EntityMap

function CHECK_GROUP_STAGE_FINAL_QUALIFICATION(source_run, classified_sources, prior_state):
  trigger:
    - prüfe in jedem Lauf zwei Bedingungen:
      1. Ist das letzte Gruppenspiel laut aktuellem offiziellem Turnierkern beendet?
      2. Enthält der Prior-State bereits group_stage_final_qualification_gate_passed=true plus vollständige Listen qualified_teams, best_third_placed_teams und eliminated_teams sowie einen vollständigen KnockoutBracketState?
    - nutze für Bedingung 1 aktuelle offizielle Spiel-/Fixture-/Standings-Quellen und mindestens eine allgemeine Websuche, damit unerwartete Änderungen erkannt werden
  if last_group_match_finished == false:
    - required = false
    - group_stage_final_qualification_gate_passed bleibt false oder absent
    - markiere alle noch offenen Drittplatzierten-/Qualifikationsfragen als provisional
    - keine endgültige Eliminierung/Qualifikation allein aus Drittplatz möglich
    - returns QualificationGateResult
  if last_group_match_finished == true and prior_state.group_stage_final_qualification_gate_passed == true:
    required actions:
      - vollständige Listen und KnockoutBracketState aus dem Prior-State übernehmen:
        qualified_teams
        best_third_placed_teams
        eliminated_teams
        knockout_bracket_state
      - prüfen, ob aktuelle offizielle Quellen eine nachträgliche Korrektur, Disziplinarentscheidung oder Bracket-Änderung melden
      - wenn keine offizielle Korrektur vorliegt: passed = true, reused_from_prior_state = true, no_full_recalculation_required = true
      - wenn eine offizielle Korrektur vorliegt: Gate erneut öffnen und vollständige Prüfung ausführen
    returns QualificationGateResult
  if last_group_match_finished == true:
    required actions:
      - offizielle Endtabellen aller Gruppen abrufen und dokumentieren
      - direkte Qualifikanten je Gruppe bestimmen
      - alle Drittplatzierten erfassen
      - beste Drittplatzierte nach offiziellen Wettbewerbs- und Tie-Break-Regeln bestimmen
      - offizielle K.o.-Runden-/Bracket- oder Qualifikantenliste prüfen, sobald verfügbar
      - Konflikte zwischen Tabellen, Bracket und Newsquellen als blocking_conflict behandeln
      - vollständige Liste erstellen: qualified_teams, eliminated_teams, best_third_placed_teams
      - jede Änderung des Teamstatus mit Quelle, Datum, Regelgrund und Kausalkette dokumentieren
      - im neuen Prior-State speichern:
        group_stage_final_qualification_gate_passed=true
        group_stage_final_qualification_gate_checked_at=RUN_TIMESTAMP
        group_stage_final_qualification_sources=[...]
        qualified_teams=[...]
        best_third_placed_teams=[...]
        eliminated_teams=[...]
        KNOCKOUT_BRACKET_STATE as serialized KnockoutBracketState with bracket_locked, current_stage, sources, rounds and team_bracket_positions
        qualification_gate_reopen_only_on_official_correction=true
    required evidence:
      - mindestens eine aktuelle offizielle T0/FIFA-Quelle für Tabellen/Spielstatus/Bracket
      - falls offizielle Bracket-Liste noch fehlt: offizielle Gruppentabellen plus dokumentierte Tie-Break-Regelberechnung
      - keine Such-Snippets als alleinige Qualifikationsquelle
    pass criteria:
      - all_group_matches_complete == true
      - knockout_qualified_teams_complete == true
      - knockout_bracket_state_complete == true
      - best_third_placed_verified == true
      - eliminated_teams_verified == true
      - no_blocking_qualification_conflict == true
      - prior_state_will_persist_qualification_gate == true
    if any pass criterion false:
      - passed = false
      - output_status must be NOT_CURRENT or DEGRADED_NOT_PUBLISHABLE according to severity
      - mark the run as not publishable; downstream publishing must not run
    returns:
      QualificationGateResult

function UPDATE_KNOCKOUT_BRACKET_STATE(source_run, classified_sources, prior_state, group_stage_gate):
  required actions:
    - Wenn group_stage_gate nicht bestanden oder Gruppenphase nicht abgeschlossen ist: bracket_locked=false setzen und keine endgültigen K.o.-Pfade erzwingen.
    - Wenn prior_state.knockout_bracket_state vorhanden und bracket_locked=true ist: diesen Baum als Ausgangspunkt übernehmen.
    - In jedem Lauf aktuelle offizielle Quellen auf neue Endstände, Forfait-/Disziplinarentscheidungen, Bracket-Korrekturen und Spielverlegungen prüfen.
    - Neue bestätigte K.o.-Ergebnisse in rounds aktualisieren: status=completed, score, winner, loser_eliminated=true, winner_to-Slot befüllen, Verliererstatus eliminieren.
    - Noch offene Paarungen eindeutig als team_a/team_b oder winner_match_id-Platzhalter speichern; keine freien Textpfade.
    - current_stage aus dem frühesten noch offenen Match bestimmen.
    - display_intermediate_probability bestimmen:
      * quarter_final_probability, wenn mindestens ein Achtelfinal-/R16-Platz zur QF-Erreichung noch offen ist.
      * semi_final_probability, wenn der Viertelfinalpfad die nächste relevante Zwischenrunde ist und das Finale noch nicht als einzige offene Partie übrig ist.
      * none, wenn nur noch das Finale offen ist.
    - team_bracket_positions für jedes noch lebende Team aktualisieren: half, quarter, slot, current_match_id, possible_next_match_id.

function BUILD_MATCHUP_MATRIX(team_states, pre_match_dossiers):
  required actions:
    - paarweise Matchups berechnen
    - Kontextnachteile berechnen
    - Shock-Vorteile berechnen
    - Grundwahrscheinlichkeit aus Teamzustand und festem Turnierpfad berechnen

function NORMALIZE_PROBABILITIES(path_result, bracket_state):

    - quarter_final_probability_percent auf 800.0 prüfen bzw. normalisieren, solange der Wert intern relevant ist

function BUILD_MAIN_PROBABILITY_TABLE(team_states, prior_state, current_event_chains):
  required actions:
    - Sortiere aktive Einzelteams nach title_probability absteigend.
    - Teams mit title_probability 0.0 dürfen nur dann in der Haupt-Wahrscheinlichkeitstabelle erscheinen, wenn sie im aktuellen Lauf gerade auf 0.0 gefallen bzw. gerade ausgeschieden sind.
    - Teams, die bereits in einem früheren Lauf auf 0.0 gefallen waren, werden aus der Haupttabelle entfernt.
    - Entfernte 0.0-Teams bleiben im Prior-State nachvollziehbar erhalten: eliminated_teams, knockout_eliminated_teams_after_prior_cutoff oder ein geeigneter Archiv-/Statusabschnitt.
    - Wenn Restfeld eine positive Titelwahrscheinlichkeit hat, muss die Haupttabelle genau 25 Zeilen enthalten.
    - Freie Tabellenplätze müssen mit noch aktiven Teams gefüllt werden, die fachlich im Restfeld-Aggregat enthalten sind.
    - Ausgeschiedene Teams aus dem Restfeld dürfen nicht als Füllteams erscheinen; sie bleiben nur in Status-/Archivabschnitten nachvollziehbar.
    - Restfeld-Füllteams nach internem Restfeld-Wert absteigend auswählen; wenn diese Werte im Prior-State nicht bestimmt sind, muss der Lauf sie aus Restfeld-Signalen, K.o.-Pfad, Qualifikationsstatus, Gegnerpfad, Teamstärke-/Indexhinweisen und aktuellen Quellen nachvollziehbar ermitteln.
    - Restfeld-Füllteams sind keine reinen Anzeigezeilen: Wenn sie in der Haupttabelle erscheinen, müssen title_probability_percent, final_probability_percent, quarter_final_probability_percent, semi_final_probability_percent, early_exit_probability_percent und crash_risk_score vollständig modelliert werden.
    - Dasselbe gilt für eine verbleibende Restfeld-Restzeile als Aggregat für Title, Final und die aktuell angezeigte Zwischenrunde. early_exit_probability_percent und crash_risk_score bleiben bei der Restfeld-Rest-Aggregatzeile leer, solange sie nicht ausdrücklich als gewichtete Aggregatindikatoren mit eigener Kennzeichnung definiert werden.
    - Für Restfeld-Füllteams und Restfeld-Restzeile dürfen keine Werte erfunden oder doppelt gezählt werden: Summe der Füllteam-Titelwerte plus Restfeld-Restzeile muss exakt der bisherigen Restfeld-Gesamtwahrscheinlichkeit entsprechen.
    - Final-/Halbfinal-/Viertelfinal-/Early-Exit- und Crash-Risk-Werte müssen aus derselben Pfadsimulation bzw. derselben Restfeld-Komponentenrechnung stammen; wenn das nicht möglich ist, gilt der Lauf als nicht erfolgreich und darf nicht finalisiert werden.
    - Wenn Restfeld-Komponenten neu aus dem Aggregat gehoben werden, muss geprüft werden, ob die Pfadsimulation dadurch andere Teams beeinflusst; nötige Korrekturen an anderen Teams müssen mit Kausalkette dokumentiert werden.
    - Die Restfeld-Restzeile steht immer am Ende der Haupttabelle, solange ein positiver Restfeld-Rest verbleibt.
    - Wenn kein positives Restfeld mehr existiert, wird keine Restfeld-Zeile angezeigt und die Haupttabelle darf weniger als 25 Zeilen haben.
    - Ränge nach dieser Anzeige-/Lebenszyklusregel lückenlos neu berechnen.
    - Probability-Summe bleibt über modellierte Einzelteams plus Restfeld-Füllteams plus Restfeld-Restzeile 100.0.
    - Title % und Final % bleiben feste Tabellenspalten.
    - Die Zwischenrunden-Spalte wird aus probability_result.display_intermediate_probability gewählt:
      * quarter_final_probability -> ausschließlich Tabellenkopf "QF %" und Summe 800.0; HF % nicht anzeigen.
      * semi_final_probability -> ausschließlich Tabellenkopf "HF %" bzw. "Semi-final %" und Summe 400.0; QF % vollständig aus der HTML-Haupttabelle entfernen.
      * none -> keine Zwischenrunden-Spalte anzeigen, wenn nur noch das Finale offen ist; insbesondere weder QF % noch HF % anzeigen.
    - Das Anzeige-Gate ist exklusiv: In der HTML-Haupttabelle darf nie mehr als eine Zwischenrunden-Spalte gleichzeitig erscheinen. Frühere Rundenwerte bleiben ausschließlich für Audit und Verlustfreiheit im Prior-State erhalten.
    - Summenprüfungen: Summe title_probability_percent = 100.0; Summe final_probability_percent = 200.0; wenn QF % angezeigt wird, Summe quarter_final_probability_percent = 800.0; wenn HF % angezeigt wird, Summe semi_final_probability_percent = 400.0. Diese Regeln gelten über die gesamte Haupttabelle inklusive Restfeld-Komponenten und Restfeld-Rest-Aggregat.
    - Monotonieprüfung pro Team: title_probability_percent <= final_probability_percent <= semi_final_probability_percent <= quarter_final_probability_percent und kein Rundenwert > 100.0.
    - crash_risk_score wird nicht aufsummiert; für Einzelteams ist er individuell, für eine Restfeld-Rest-Aggregatzeile bleibt er leer oder wird ausdrücklich als gewichteter Aggregatindikator markiert.
    - early_exit_probability_percent wird für Einzelteams modelliert; für eine Restfeld-Rest-Aggregatzeile bleibt er leer oder wird ausdrücklich als gewichteter Aggregatindikator markiert.
  returns:
    MainProbabilityTable

function FINALIZE_RUN(context, artifacts, artifact_audit, model_audit = none):
  success criteria:
    - source_run.completed == true
    - context.output_status is CURRENT or DEGRADED
    - model_audit.passed == true when model_audit exists
    - artifact_audit.passed == true
  if success criteria all true:
    - mark the four main artifacts as successful and ready for the daily-run orchestrator
    - do not create downstream publishing artifacts inside the model run
    - downstream publishing and deployment are handled only by the daily-run orchestrator after this success result
  else:
    - mark the four main artifacts as failed and include a short failure reason for the daily-run orchestrator
    - do not create downstream publishing artifacts inside the model run
  returns:
    FinalizationResult

GATE_CURRENT_ALLOWED:
  pass if:
    source_run.completed == true
    source_quality.total_score >= 75
    unresolved_blocking_conflicts empty
    critical_missing_data empty
    t0_core_checked == true

GATE_DEGRADED_ALLOWED:
  pass if:
    source_run.completed == true
    source_quality.total_score >= 60
    unresolved_blocking_conflicts empty
    critical_missing_data empty

GATE_NOT_CURRENT_REQUIRED:
  true if:
    source_run.completed == false
    or source_quality.total_score < 60
    or unresolved_blocking_conflicts not empty
    or critical_missing_data not empty
    or t0_core_checked == false

GATE_PROBABILITIES_ALLOWED:
  pass if:
    output_status in ["CURRENT", "DEGRADED"]
    probability_result.normalization_passed == true
    probability_result.bracket_constraints_passed == true
    probability_sum between 99.5 and 100.5
    final_probability_sum between 199.5 and 200.5
    if display_intermediate_probability == quarter_final_probability: quarter_final_probability_sum between 799.5 and 800.5
    if display_intermediate_probability == semi_final_probability: semi_final_probability_sum between 399.5 and 400.5
    audit.passed == true

GATE_FINAL_DELIVERY_ALLOWED:
  pass if:
    audit.passed == true
    artifact_audit.passed == true
    html download link exists
    prior-state download link exists
    changes txt download link exists

7. OUTPUT-STATUS
============================================================

CURRENT:
  Aktuelle Quellen vorhanden.
  Quellenqualität mindestens 75.
  Keine blockierenden Datenlücken.
  Nicht blockierende Unsicherheiten sichtbar.

DEGRADED:
  Aktuelle Quellen vorhanden.
  Quellenqualität mindestens 60.
  Keine blockierenden Datenlücken.
  Konfidenz reduziert.

NOT_CURRENT:
  Aktueller Quellenlauf fehlt.
  Oder Quellenqualität unter 60.
  Oder T0-Kern fehlt.
  Oder blockierender Konflikt offen.
  Oder kritische Daten fehlen.

CURRENT_WITH_NO_NEW_MODEL_EVENTS:
  Aktuelle Quellen wurden geprüft.
  Keine belastbaren neuen modellrelevanten Events seit dem Prior-State.
  Zustand bleibt unverändert.
  Keine künstlichen Deltas.
